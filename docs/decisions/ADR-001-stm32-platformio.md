# ADR-001: Use STM32F446RE with STM32CubeIDE for Motor Control Firmware

**Date:** 2026-05-03
**Status:** Accepted
**Deciders:** Software Team Lead, Project Founder

---

## Context

We need a microcontroller to run the real-time CAN bus control loop for the AK40-10 motors. The controller must:
- Generate CAN frames at 500-1000 Hz without missing deadlines
- Communicate with a Windows PC over serial for development and debugging
- Be accessible to team members with varying experience levels
- Have reliable, well-documented CAN hardware support

## Decision

Use the **STM32F446RE** (on the ST Nucleo F446RE development board) programmed with **STM32CubeIDE**, using HAL-generated C code.

## Rationale

| Option | Pros | Cons |
|---|---|---|
| STM32F446RE + CubeIDE | Industry-standard MCU, hardware bxCAN, 180 MHz Cortex-M4 with FPU, HAL well-documented, familiar to electrical engineers, ST-Link on Nucleo | Steeper learning curve than Arduino |
| STM32 + PlatformIO/Arduino | Easier Arduino API, good CI support | Different pin defaults, diverges from HAL-based workflow used for Phase 1 and 2 tasks |
| ESP32 + PlatformIO | Built-in WiFi, large community | TWAI peripheral less mature, weaker real-time guarantees |

CubeIDE was chosen because:
- The founder's hardware sessions and task specifications target CubeIDE + HAL
- HAL is the standard approach for STM32 in industry and academia
- CubeMX handles peripheral configuration visually, reducing setup errors
- Consistent with what the electrical engineers on the team already know

## Hardware configuration

- **Board:** ST Nucleo F446RE
- **Serial (PC comms):** USART2 -- PA2 (TX), PA3 (RX) via ST-Link virtual COM, 115200 baud
- **CAN peripheral:** bxCAN (CAN1) -- PA11 (RX), PA12 (TX)
- **External CAN transceiver required:** SN65HVD230 or TJA1050 (Electrical team)
- **CAN baud rate:** 1 Mbit/s

**CAN timing registers for 1 Mbit/s at 90 MHz APB1:**

| Register | Value |
|---|---|
| Prescaler | 9 |
| Time Seg 1 | 6 |
| Time Seg 2 | 3 |
| SJW | 1 |

## Consequences

- All firmware is written in C targeting the STM32 HAL (generated by STM32CubeIDE / CubeMX).
- The ak40_driver library (ak40_driver.h / ak40_driver.c) is written in C and uses HAL CAN directly.
- USART2 via ST-Link virtual COM is used for serial -- no USB CDC, no enumeration delay.
- WiFi is not available on the Nucleo. Wireless communication is deferred to Phase 5+.
- CI firmware build check is removed since CubeIDE does not have a simple headless CLI. Firmware is verified locally by the developer.
